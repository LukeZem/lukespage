Lucas. Spring 2022. Cybersecurity and Information Assurance, Hack This Site

This artifact is a short summarization of an assignment utilizing “Hack this site” to learn and employ practical hacking techniques. It details the student’s experience with the project and how difficult or enjoyable certain parts were.

I chose this artifact because I really enjoyed this assignment, and this class is what steered my interest in computer science to a cyber security direction. It also introduced me to a great site called “hackthissite.com.” A place where you can legally practice real world hacking techniques and in the process learn what techniques attackers use to gain unauthorized access, information, or just probe defenses. This artifact, or maybe more so the exercise itself, demonstrated my ability to utilize information security practices by showing I know, at least in part, how malicious actors can operate.

Zemlin, Lucas. Spring 2022. Cybersecurity and Information Assurance, IA Policy Draft

This artifact is a draft of just 2 sections of an information assurance policy that could potentially be used by an organization. It contains rules and policy information about good email usage practices, as well as what’s called clean station practices.

I enjoyed writing this draft very much because it made me think critically and practically about what standards and practices would be enforced in a real-world organization. I had to apply the skills and knowledge I had learned in CSS 310 to come up with ideas that would benefit an organization I might potentially work for in the future. Writing IA policy’s is a large part of working in cybersecurity. As regular end users are often not aware how attackers operate and gain access. In fact, in my current job as an IT – Information security intern, I regularly come up with policy drafts for topics such as passphrase creation, MFA usage, and other company wide security mandates. This project helped give me critical knowledge to be successful in those endeavors.

Zemlin, Lucas. Summer 2022. CSS 475 Databases, Final Programming Project

This project is a compilation of works completed over the course of a quarter. It is a simple database created using MySQL and then a basic webpage built using PHP to be able to interact with the database. It demonstrates a basic knowledge of PHP and database creation using MySQL.

This project is important because databases are an integral part of software development and information technology. Understanding how databases are built and how APIs are used to push, pull, and update data within them are fundamental to any job in IT or cybersecurity. This was a team effort, so I got to see what it was like working in a mock software team. Working all the way from the conceptual ideation of the database on paper, to writing the SQL statements that create the database, to using PHP to generate a webpage and create backend functionality that allowed interaction with it.
